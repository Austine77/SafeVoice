SafeVoice 2026 update completed
================================

Folder names preserved:
- backend folder remains: backend
- frontend folder remains: worldwide
- No project folder was renamed.

Main fixes applied:
1. Fixed backend static page serving bug in backend/server.js.
   - The helper was named sendfworldwideFile but routes called sendWorldwideFile.
   - This would make /, /staff.html, /police-contact.html, and /social-worker-contact.html fail at runtime.

2. Updated backend CORS origin loading.
   - Backend now reads CLIENT_URL, WORLDWIDE_URL, FRONTEND_URL, APP_BASE_URL, and CORS_ORIGINS.
   - This helps Render frontend and backend connect correctly.

3. Updated staff police/social-worker systems to worldwide mode.
   - worldwide/police-contact.html is now a worldwide police/law-enforcement contact system.
   - worldwide/social-worker-contact.html is now a worldwide social worker/child protection contact system.
   - Both pages load country options and query backend route /api/global-directory/contacts.
   - Both pages keep login protection through SafeVoice-staff-authenticated.

4. Rebuilt worldwide/dist.
   - npm --prefix worldwide run build completed successfully.

Important contact-list note:
- backend/data/globalDirectory.json contains 1,001 records across 250 country/territory codes.
- The file has emergency, police, child_protection, and social_support categories.
- Many records are seed/placeholder records marked requires_local_verification.
- Before real police/social-worker handover in any country, verify phone numbers from official government, police, child protection, or recognized helpline sources.
